# یادداشت‌های کاری موقت

> این فایل برای نکات لحظه‌ای و کارهای نیمه‌کاره است؛ هر مورد بعد از حل‌شدن کامل از این‌جا پاک می‌شود (دستورالعمل #۱۱).

## ⚠️ روش تست درست (خیلی مهم — منشأ ساعت‌ها سردرگمی بود)

هر بار که اکستنشن را در `chrome://extensions` رفرش می‌کنید، **تمام تب‌هایی که از قبل باز بوده‌اند** content script
یتیم (orphaned) پیدا می‌کنند: آیکن هنوز دیده می‌شود ولی هر `sendMessage` با خطای `Extension context invalidated`
شکست می‌خورد. این رفتار استاندارد کروم است، نه باگ ما. **روش تست درست:**

۱. اول اکستنشن را رفرش کن. ۲. **بعد** یک تب کاملاً جدید باز کن (یا تب موجود را Reload کن) تا content script تازه‌ای
که به نسخه‌ی فعلی اکستنشن تعلق دارد تزریق شود. ۳. حالا تست کن.

هرگز در تبی که *قبل از* رفرش اکستنشن باز شده تست نکن. (کد حالا این حالت را تشخیص می‌دهد و پیام «افزونه به‌روزرسانی
شده؛ صفحه را رفرش کنید» نشان می‌دهد — رفتار درست برای تولید هم هست، چون آپدیت اکستنشن همین وضعیت را می‌سازد.)

## باز — نیاز به پیگیری

- **⏳ منتظر نتیجهٔ بازبینی Chrome Web Store** — آیتم در تاریخ ۸ ژوئیهٔ ۲۰۲۶ Submit شد؛ وضعیت **Pending Review**
  (ممکن است چند روز کاری طول بکشد). Item ID در حافظهٔ محلی ثبت شده. کارِ در جریان همین است؛ منتظر ایمیل/تغییر وضعیت.
- **اگر بازبین دربارهٔ `<all_urls>` سؤال کرد**: توجیهِ آماده در [docs/chrome-web-store-listing.md](docs/chrome-web-store-listing.md)
  و متن Test instructions هست — با هم جواب را می‌فرستیم.
- **گزینهٔ v0.1.1 (اختیاری)**: حذف `host_permissions: <all_urls>` با معماری Port — `content.ts` با
  `chrome.runtime.connect()` کانکشن باز کند و `background.ts` نتایج را به‌جای `tabs.sendMessage` روی همان port
  برگرداند؛ هندلر `commands` هم از همان port استفاده کند. سپس bump به `0.1.1`، `npm run zip`، آپلود آپدیت.
  **هشدار واقع‌بینانه**: `content_scripts` روی `<all_urls>` ذاتی محصول است و می‌ماند، پس هشدار نصبِ «all websites»
  و بازبینیِ «runs everywhere» کاملاً از بین نمی‌رود؛ فقط مجوز host_permissions و اصطکاک بازبینیِ آن کم می‌شود.

## حل‌شده‌ها (فقط برای یادآوریِ چراییِ تصمیم‌ها؛ اکشن ندارند)

- **`npm audit`**: ۸ هشدار فقط در زنجیرهٔ devDependencies ابزار build (`wxt`→`web-ext-run`)؛ با `npm audit --omit=dev`
  صفر آسیب‌پذیری در بستهٔ نهایی. در README شفاف بیان شد. عمداً `audit fix --force` زده نشد (build را می‌شکند).
- **نسخه**: `0.0.0` → `0.1.0` انجام شد.
- **i18n همه‌ی ۲۰ زبان کامل شد** (TS با `Dict = Record<MessageKey,string>` کامل‌بودن هر زبان را الزام می‌کند). زبان‌های
  کم‌مصرف‌تر best-effort‌اند و در README از اصلاحات جامعه استقبال می‌شود.
- **نقش پاپ‌آپِ زنده موجه شد**: نمایش interim، کپی/توقف، حالت popup-only، نشانگر صدا و سوییچ زبان — و حالا ⚙️ تنظیمات.
- **نام نهایی «Voice to Text»** (فارسی: «تبدیل صدا به متن») در `public/_locales/*` تثبیت شد — دیگر موقت نیست.
- **موتور ابری (OpenAI/Gemini)** ساخته و سپس **حذف شد**؛ اگر روزی خواستید برگردد، در git history هست و نکات فنی‌اش
  (WAV، VAD، فرمت درخواست‌ها) در research-notes.md § ۵ مانده است.
- **`getSettings()` هرگز throw نمی‌کند** و روی خطا `DEFAULT_SETTINGS` برمی‌گرداند — لازم چون `wxt/storage` خارج از
  context اکستنشن (پیش‌نمایش http یا content script یتیم) خطا می‌دهد.
