# نقشه‌راه پروژه — گفتارنویس (اکستنشن تبدیل صدا به متن)

> این فایل زنده است و باید همیشه با پیشرفت واقعی پروژه به‌روز بماند (دستورالعمل عملیاتی #۱۲).
> برای پس‌زمینه‌ی تصمیمات، ببینید: [docs/proposal.md](docs/proposal.md) و [docs/research-notes.md](docs/research-notes.md)

## تصمیمات تأیید‌شده با کاربر

- موتور STT: **فقط رایگان** (Web Speech API). موتور ابری در فاز ۲ ساخته شد ولی به‌درخواست کاربر **حذف شد** (دردسر
  خطاها + سنگینی). همه‌چیز رایگان و سبک.
- زبان پیش‌فرض گفتار: **انگلیسی**. رابط کاربری چندزبانه (i18n) با انتخاب زبان.
- اعداد **هوشمند به زبان گفتار** نوشته می‌شوند (فارسی→۱۲۳، عربی→٠١٢٣) — بدون گزینه.
- دستورهای نگارشی برای **همه‌ی زبان‌ها**.
- تحویل متن: **درج مستقیم در فیلد + پاپ‌آپ زنده**؛ آیکن **بیرون گوشه‌ی راست فیلد** (نه داخلش).
- **UI آیکن‌محور** (کمترین متن، بین‌المللی). تنظیمات هم در صفحه‌ی onboarding (اولین استفاده) و هم آیکن ⚙️ در پاپ‌آپ.

### درس‌های مهم این فاز (برای فازهای بعدی)

- **`runtime.sendMessage` به content script نمی‌رسد** — برای پیام extension→content-script حتماً
  `tabs.sendMessage(tabId, …)` لازم است (research-notes.md § ۱د). بزرگ‌ترین درس این فاز.
- در معماری پیام‌رسانی broadcast-based، **هرگز به پیام «درخواست از کلاینت» و پیام «دستور relay‌شده» یک `target`
  مشترک ندهید** — جزئیات کامل در research-notes.md § ۱ج.
- تب‌های باز از قبل، بعد از هر بار reload کردن اکستنشن، به خطای `Extension context invalidated` می‌خورند (رفتار
  شناخته‌شده‌ی کروم، نه باگ ما) — برای تست همیشه تب کاملاً تازه باز شود. (کد حالا این حالت را graceful مدیریت می‌کند.)
- کنسول هر context (صفحه‌ی وب، service worker، Offscreen Document) کاملاً جداست؛ برای دیباگ کامل هر سه باید جداگانه
  چک شوند.

## چطور همین حالا تست کنید (فاز ۰)

```
npm run dev
```
این دستور کروم را با extension بارگذاری‌شده باز می‌کند (یا اگر ترجیح می‌دهید build دستی: `npm run build` سپس در
`chrome://extensions` حالت Developer mode را فعال کنید → Load unpacked → پوشه‌ی `.output/chrome-mv3` را انتخاب کنید).

با اولین نصب، صفحه‌ی onboarding خودکار باز می‌شود. اگر نشد، در `chrome://extensions` روی extension کلیک کنید و از
جزئیات آن، یا مستقیم به آدرس زیر بروید (ID را از صفحه‌ی extensions بردارید):
`chrome-extension://<EXTENSION_ID>/onboarding.html`

۱. دکمه‌ی «فعال‌سازی میکروفون» را بزنید و اجازه بدهید.
۲. زبان fa-IR را انتخاب کنید، «شروع ضبط» را بزنید، چند جمله‌ی فارسی بگویید، «توقف ضبط» را بزنید.
۳. نتیجه (دقت، سرعت، مشکلات) را برای من گزارش کنید تا فاز ۱ را بر اساس آن جلو ببریم.
